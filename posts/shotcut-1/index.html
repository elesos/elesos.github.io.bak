shotcut系列1编译

elesos 发表于 2022-05-252022-05-25T22:30:00+08:00
2 分钟 阅读

有个sdk包括了所有依赖，可以试试 https://shotcut.org/notes/

mkdir build cd build cmake -DCMAKE_INSTALL_PREFIX=/usr/local/ /path/to/shotcut_source cmake –build . //编译 cmake –install . //安装

1， win 上编译 https://shotcut.org/notes/windowsdev/

2，安装 msys2 https://www.msys2.org/ 安装后运行 pacman -Syu 更新package database and base packages.

Run “MSYS2 MSYS” from Start menu，Update the rest of the base packages with pacman -Syu

现在可以安装软件了： pacman -S –needed base-devel mingw-w64-x86_64-toolchain 关闭终端，从开始菜单打开 “MSYS MinGW 64-bit” ，然后就可以调用make ,gcc了。

错误： Could NOT find PkgConfig (missing: PKG_CONFIG_EXECUTABLE) https://download.gnome.org/binaries/win64/dependencies/pkg-config_0.23-2_win64.zip https://download.gnome.org/binaries/win64/dependencies/gettext-runtime_0.18.1.1-2_win64.zip https://download.gnome.org/binaries/win64/glib/2.26/glib_2.26.1-1_win64.zip 解压拷贝里bin的相关文件（intl.dll ，libglib-2.0-0.dll，这2个dll，pkg-config用到，如果提示覆盖intl.dll，备份一下后覆盖 ）到 C:\MinGW\bin

  • No package ‘fftw3’ found vcpkg.exe install fftw3:x64-windows 然后加上 -DCMAKE_TOOLCHAIN_FILE=F:\vcpkg\scripts\buildsystems\vcpkg.cmake

  • CMake Error in src/CMakeLists.txt: Imported target “PkgConfig::mlt++” includes non-existent path

    “D:/a/shotcut/shotcut/build/Shotcut/include/mlt-7/mlt++”

修改C:\Projects\Shotcut\lib\pkgconfig\mlt++-7.pc C:\Projects\Shotcut\lib\pkgconfig\mlt-framework-7.pc改成C:/Projects/Shotcut用反斜杠 删除CMakeCache.txt文件 把cmake生成的输出目录也删除

  • CMake Error in src/CMakeLists.txt: Imported target “PkgConfig::FFTW” includes non-existent path

    “F:/vcpkg/packages/fftw3_x64-windows/debug/include” 把F:/vcpkg/packages/fftw3_x64-windows目录下的Include拷贝进去即可。

开始编译！

  • undefined reference to `__imp__ZNK7QStringeqE13QLatin1String’ CMAKE_PREFIX_PATH 里面添加上qt mingw路径

set(defQtdir “E:/Qt/5.15.2/5.15.2/mingw81_64”) list (APPEND CMAKE_PREFIX_PATH ${defQtdir}) 安装qt时不要安装cmake, ninja,mingw

看见 – Configuring done – Generating done 就说明配置好了。

下一步点运行。

E:/Qt/6.2.1/Tools/mingw810_64/bin/../lib/gcc/x86_64-w64-mingw32/8.1.0/../../../../x86_64-w64-mingw32/bin/ld.exe: i386 architecture of input file `CMakeFiles\shotcut.dir/objects.a(shotcut.rc.obj)’ is incompatible with i386:x86-64 output 应该是版本不对，去掉path里面的E:\Qt\6.2.1\Tools\mingw810_64\bin 还有QTDIR也要修改，不行就把6.2.1的qt目录改下名字，还要设置下qt creator的qt version

可以正常调试了。在qt creator 里面直接点运行，可以跑起来。如果不能启动,就把安装目录下面的相关文件都拷贝到shotcut.exe所在目录.

但还不能在目录下双击运行。 Where to add “make install” in Qt Creator? 在Build steps里面可以勾选install

会安装qml文件（ QML files reads at run-time.） 将安装的Program Files目录下的文件拷贝到shotcut.exe所在目录后，并拷贝了几个库之后，双击可以正常运行。
